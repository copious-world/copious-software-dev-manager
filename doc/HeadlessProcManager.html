(async) app_publication_pre_fan_response(topic, msg_obj) → {boolean}

Description:

  • Runs before writing publications.

    This runs if app_can_block_and_respond. A true value returned means that the application has blocked the publication.

    Gives the application the chance to refuse a publication or to pass it based on criterea.

    The application may take care of any operation it requires for publication prior to publishing with the intention of returning false to pass this on to publication. (One might imagine a situation where the application will read a measurement from sensor and publish the message if the sensor has actually changed. The application version of this method would return false indicating that the update publication will not be blocked. The method can be used to write the new value onto the message object.)

    The pulication method, send_to_all awaits the return of this method.
